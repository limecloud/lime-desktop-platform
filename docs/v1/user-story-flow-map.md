# 用户流映射

## 1. 目的

把平台能力映射到真实用户动作，避免只写模块、不写路径。

## 2. 主要角色

| 角色 | 目标 | 主要入口 |
| --- | --- | --- |
| 普通用户 | 打开已安装 App 并开始使用 | 应用中心 / 最近使用 / 入口卡片 |
| App 管理员 | 安装、更新、启用、禁用 App | 应用中心 / 应用详情 |
| 平台管理员 | 配置模型、OAuth、OEM、充值和代理 | 设置中心 |
| 开发者 | 检查 manifest、readiness 和 bridge 日志 | 开发者页 |
| OEM 运营 | 切换品牌、渠道和展示文案 | OEM 设置页 |

## 3. 主流用户流

### 3.1 第一次打开平台

```text
启动桌面壳层
-> 读取品牌 bootstrap
-> 检查 OAuth / 会话
-> 拉取应用中心目录
-> 展示应用列表
-> 进入 App 或设置页
```

成功标准：

- 用户看见品牌。
- 用户看见当前可用 App。
- 未登录时能进入登录流程。

### 3.2 安装一个新 App

```text
在应用中心选择 App
-> 查看详情
-> 下载 / 校验 / projection / readiness
-> 补齐设置或授权
-> 启用入口
-> 启动 App
```

成功标准：

- 安装前能看见版本和来源。
- blocked 原因能显示。
- ready 才能启动。

### 3.3 更新已有 App

```text
平台检查更新
-> 显示可更新版本
-> 下载新包
-> 校验 hash
-> 重新 projection 和 readiness
-> 切换版本
```

成功标准：

- 用户知道当前版本和新版本差异。
- 更新失败不会破坏旧版本。

### 3.4 处理 needs-setup

```text
App 显示 needs-setup
-> 进入修复面板
-> 补 OAuth / 模型 / OEM / 充值
-> 重新检查 readiness
-> 回到 ready
```

成功标准：

- 修复动作明确。
- 用户不需要猜原因。

### 3.5 管理设置

```text
打开设置中心
-> 修改模型 / OAuth / OEM / 充值 / 代理
-> 保存
-> 重新投影到运行中的 App
```

成功标准：

- 设置变化能被当前壳层感知。
- 运行中的 App 能收到新 snapshot。

### 3.6 开发者诊断

```text
打开开发者页
-> 查看 manifest
-> 查看 projection
-> 查看 readiness
-> 查看 bridge 日志
-> 定位 blocked 原因
```

成功标准：

- 问题可复现。
- 阻断原因可解释。

## 4. 页面到动作

| 页面 | 主动作 | 次动作 |
| --- | --- | --- |
| 应用中心 | 安装 / 启动 | 查看详情 / 更新 |
| 应用详情 | 启动 / 启用 | 禁用 / 查看版本 |
| 设置中心 | 保存配置 | 切换子页 |
| 运行页 | 查看事件流 | 返回应用中心 |
| 开发者页 | 诊断 / 导出日志 | 复制契约信息 |

## 5. 更多主路径

### 5.1 首次启动

```text
打开平台
-> 读取品牌 bootstrap
-> 读取 OAuth 会话
-> 读取应用中心目录
-> 进入应用中心或默认入口
```

### 5.2 安装 App

```text
在应用中心选择 App
-> 查看 manifest / 版本 / 来源
-> 下载或校验本地包
-> 生成 projection
-> 计算 readiness
-> 进入 ready 或显式阻断
```

### 5.3 启动 App

```text
点击启动
-> Host Runtime 初始化
-> 注入 host snapshot
-> App 进入运行页
-> 调用桥接能力
```

### 5.4 修改共享设置

```text
进入设置中心
-> 修改模型 / OAuth / OEM / billing / 平台设置
-> 保存
-> 回写投影和运行中的 snapshot
```

### 5.5 开发者排查

```text
进入开发者页
-> 查看 manifest
-> 查看 projection
-> 查看 readiness
-> 查看 bridge 日志
-> 修复 blocking 条件
```

## 6. 异常路径

| 场景 | 用户应该看到什么 | 下一步 |
| --- | --- | --- |
| 未登录 | 未认证状态和登录入口 | 去 OAuth 页面 |
| 模型缺失 | 待配置提示 | 去模型设置页 |
| 品牌缺失 | 品牌待补齐 | 去 OEM 设置页 |
| billing 异常 | 需要处理 | 去充值 / 订阅页 |
| 包校验失败 | 阻断原因 | 重新安装或更新 |
| bridge 失败 | 调用失败和日志入口 | 去开发者页 |

## 7. 复用故事

| 复用对象 | 目标 | 验收要点 |
| --- | --- | --- |
| `content-studio` | 作为首个样板 App | 能在不复制登录和设置逻辑的情况下接入 |

| 后续 OEM App | 作为泛化客户 | 只需要替换 manifest、品牌和少量入口 |
